# asana-cli Rust Port Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** 既存の `asana-oauth-cli` をベースに、個人利用向け Asana OAuth CLI の Rust 版 `asana-cli` を実装し、GitHub Releases で macOS / Linux 向けバイナリを配布できる状態にする。

**Architecture:** 単一 crate の Rust CLI として実装し、`cli` / `config` / `oauth` / `oauth_callback` / `asana_api` に責務分離する。CLI は `clap`、HTTP は `reqwest`、設定保存は `serde` + JSON、localhost OAuth callback は最小限の `tokio::net::TcpListener` で処理する。テストは単体テストと `assert_cmd` ベースの統合テストで主要 UX を固定する。

**Tech Stack:** Rust 1.94+, clap, anyhow, serde, serde_json, tokio, reqwest, tempfile, assert_cmd, wiremock
